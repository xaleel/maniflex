# Contributing

## Prerequisites

1. [Install Go](https://go.dev/doc/install).
2. Download the sources and switch the working directory:

    ```bash
    go get -u -d github.com/xaleel/maniflex
    cd $GOPATH/src/github.com/xaleel/maniflex
    ```

## Submitting a Pull Request

A typical workflow is:

## Submitting a Pull Request

1. [Fork the repository.][fork]
2. [Create a topic branch.][branch] off `main`.
3. Add a failing test that captures the change.
4. Run the test and watch it fail. Because this is a multi-module
   workspace, `go test` alone won't cross module boundaries — use the
   workspace script: `bash scripts/test-all.sh` (or `scripts/test-all.ps1`
   on Windows), or `go test ./...` within the module you're editing.
5. Implement the change until the suite passes.
6. Run `go vet ./...` and `goimports -w .` (or `gofmt -w .`) to match Go
   formatting conventions.
7. Update docs and `CHANGELOG.md` if the change is user-facing.
8. [Add, commit, and push your changes.][git-help]
9. [Submit a pull request][pull-req]. Keep it focused and link any
   related issue.

[fork]: https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/fork-a-repo
[branch]: https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-branches
[git-help]: https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository
[pull-req]: https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests

## AI policy

If your contribution is fully or mostly generated by AI, you must mention so, including
the specific model used (name, version, reasoning/effort).
